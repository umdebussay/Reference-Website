6.6 Scope

Objectives

  • To be able to identify the scope of a variable

Notes

Scope

Variables have to be defined before they exist: var text;. However, just because you define a variable in one part of your code, it doesn't necessarily exist at another point in your code. This is the idea of scope. The scope of a variable is where it exists. The scope of a variable is determined by what level of {} it is defined in.

Global scope vs. local scope

Variables with a global scope exist everywhere in the script. To have global scope, variables must be defined in the root of the document (not inside a function)

Variables with local scope only exist inside the {} of the function in which they were created. So, a variable defined inside one function doesn't exist inside another function. That is why you can have two different variables with the same name in different functions and why if you create a variable inside a function, you can't refer to it from another function. Parameters are local variables. 

// Root level of document

// myGlobalVar doesn't exist yet because it hasn't been declared
// The following line would produce a reference error;
console.log(myGlobalVar);

// Variables declared at the root level have global scope
var myGlobalVar = "I exist everywhere after this line";

// Variables declared insided functions have local scope - only exist inside the function
function myFunction() {
  // Local variable
  var myLocalVar = "I only exist in this function after this line";
}

// The following line would produce an error because myLocalVar was declared inside myFunction()
console.log(myLocalVar);
