Abstraction

Abstraction in programming is the idea of removing the complicated details from something so we can think about it in terms of simpler, bigger pieces.

moveRight();
Is easier to think about than
var e = document.getElementById("block");
e.style.left += 10;
even though they accomplish the same things. This is essentially the idea of abstraction in programming.

Functional Programming

So far, most of the programming we have been doing is what is called procedural or functional programming.
In this style of programming, the program script is executed from top to bottom inside a "main" function with some other helper functions.

Functional Code Example

var num1 = prompt("Enter a start number");
var num2 = prompt("Enter an end number");
findSum();

function findSum(){
  var sum = 0;
  for (var i = num1; i <=num2; i++) {
    sum += i;
  }
  alert("The sum from " num1 + " to " + num2 + " is " + sum);
}

Object-oriented programming

Object-oriented programming is the idea of creating data structures in our programs that represent “objects” in the common sense of the word - “things”. Programs still have a main function and helper functions, but much of the program data and execution is handled within these objects. Objects increase the level of abstraction ultimately making it easier to think about how to build the program Object-oriented programming is modern programming

Objects - the concept

Right now, we can store data using specific data types: strings, integers, floats, and Booleans. It works, but it's hard to think about because it's not how we naturally think about the world. We think about the world in terms of whole “things” and then we can think about the pieces that make up the “things” and the actions that the “things” can take. This is exactly how object-oriented programming works. Objects are “things” and a new data type!

Objects

Objects have two basic parts:

  1. The data or information that describes the object (variables)
  2. The actions that the object can take (functions)

Defining JavaScript objects

In JS, object variables are called properties and the object actions are functions referred to as methods.
Objects are a new variable data type and must be defined (created) first, like any other variable. 

Object definition syntax
var objName = {
  property1: value,
  property2: value,
  method1: function() {
    // function code
  }
}
        
            
Object definition example
var player = {
  name: "Nathan",
  health: 100,
  damage: 10,
  attack: function() {
    // code to attack
  },
  takeDamage: function() {
    // code to take damage
  }
}

"this" keyword

Properties in an object are essentially the same as variables declared using var, except they are “contained” within the object. This means, you can't refer to the properties without referring to object itself ex. player.name. In order to access the object properties to use or change their values from methods inside the object, you must use the keyword this to refer to the object.

Example 

var player = {
  health: 100,
  takeDamage: function() {
    this.health -= 5;
  }
}

Accessing JavaScript object properties and methods

After an object is defined, you can use the following syntax to access the object's properties and methods. 

Property access syntax
objName.propertyName
or
objName["propertyName"]
(two different ways of doing the same thing)

Method access syntax
objName.methodName();

            
Property access example
player.name = "Jason";
player["name"] = "Norman";
alert(player.name);
var hp = player.health;

Method access example
player.attack();
